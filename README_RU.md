
# import TalkerWithAGirl

## 📌 Описание

`TalkerWithAGirl` — Python-модуль для автоматизации общения в Telegram с использованием AI (Groq API + LLaMA).

Программа анализирует входящие сообщения, строит контекст диалога и генерирует ответы в стиле живого общения.

---


# 📖 Инструкция по использованию TalkerWithAGirl

## 1. Установка зависимостей

```bash
pip install telethon aiohttp aiosqlite PyQt6
````

---

## 2. Регистрация и получение ключей

Перед первым запуском нужно зарегистрироваться:

📱 Telegram API
1. Перейди на сайт:
  https://my.telegram.org
2. Войди в аккаунт
3. Перейди в API development tools
4. Получи:
- API_ID
- API_HASH

🤖 Groq API (AI ключ)
1. Перейди на сайт:
  https://console.groq.com
2. Зарегистрируйся / войди
3. Создай API Key
4. Скопируй ключ для использования в приложении

---

## 3. Первый запуск и авторизация

1. Запусти приложение
2. Введи `API_ID` и `API_HASH`
3. Пройди авторизацию Telegram
4. После входа создастся файл сессии:

```text id="sess01"
session.session
```

---

## 4. Настройка конфигурации

Все настройки делаются прямо в приложении:

Открой ⚙️ **Settings** и заполни:

* API ID
* API HASH
* Groq API Key
* Описание себя (MY_BIO)
* PROMPT_INTRO (режим начала диалога)
* PROMPT_CHAT (режим общения)

👉 После сохранения всё применяется автоматически

---

## 5. Запуск бота
Нажми кнопку:

```text id="run01"
Connect
```

После запуска бот:

* подключается к Telegram
* начинает слушать сообщения
* отвечает автоматически через AI

---

## 6. Добавление чатов
Нажми **Add Chat** и подтверди добавление

---

## 7. Управление чатами

Для каждого чата доступно:

* ON / OFF — включение бота
* 1 / 2 — режим общения
* LAN — язык (RU / EN)
* ⋮ — дополнительные действия

---

## ⚠️ Важно

* без Groq API Key бот не отвечает
* без Telegram сессии работа невозможна
* интернет обязателен
* удаление session = повторный логин

# ⚙️ Как работает

### Авторизация Telegram

Используются:
- `API_ID`
- `API_HASH`
- `TelegramClient (Telethon)`

После первого запуска создаётся файл:

session.session

Он хранит авторизацию Telegram-сессии.

---

### Обработка сообщений

При получении нового сообщения:

* проверяется, что чат приватный
* сообщение сохраняется в SQLite
* очищается от лишних символов
* определяется язык (RU / EN)
* обновляется контекст диалога

---

### База данных

SQLite используется для хранения:

**profiles**

* user_id
* username
* name
* status (active / paused)
* bot_mode
* language
* hobbies

**messages**

* chat_id
* sender_id
* text
* timestamp

---

### AI генерация ответа

Используется Groq API:

```python
model = "llama-3.1-8b-instant"
```

В запрос передаётся:

* биография пользователя
* история диалога
* хобби собеседника
* язык общения
* выбранный режим

---

### Логика ответа

1. Получение сообщения
2. Сохранение в БД
3. Обновление контекста
4. Анализ интересов (если есть ключевые слова)
5. Формирование промпта
6. Запрос к AI
7. Разбиение ответа на части
8. Отправка в Telegram

---

### Извлечение хобби

Если сообщение содержит триггеры:

```
люблю / нравится / занимаюсь / увлекаюсь
```

AI возвращает:

```json
["спорт", "музыка", "кодинг"]
```

Данные сохраняются в профиль пользователя.

---

## 🧠 Режимы работы (пользовательские)

В этой системе **нет фиксированных режимов**.

Пользователь сам задаёт их через конфигурацию:

```json
"PROMPT_INTRO": "..."
"PROMPT_CHAT": "..."
```

### 🔹 PROMPT_INTRO

Используется для начала диалога:

* задаёт стиль первого контакта
* определяет поведение AI в начале общения

### 🔹 PROMPT_CHAT

Используется в активной переписке:

* стиль регулярного общения
* тон, краткость, манера ответа

👉 Фактически режимы = это пользовательские промпты, а не жёстко заданная логика.

## 📡 Ошибки

### Groq API

* `ERR_GROQ_401` — неверный API ключ
* `ERR_GROQ_403` — доступ запрещён
* `ERR_GROQ_429` — превышен лимит запросов
* `ERR_GROQ_500` — ошибка сервера
* `ERR_GROQ_CONNECTION` — нет соединения

---

### Telegram

* `ERR_TELEGRAM_TIMEOUT` — таймаут подключения
* `ERR_CONNECT_FAIL` — ошибка клиента

---

### AI / обработка

* `ERR_JSON_PARSE` — ошибка разбора ответа
* `ERR_RESPONSE_FAIL` — AI не смог сгенерировать ответ
